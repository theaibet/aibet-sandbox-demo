AI-Bet v3.6 → v5.0 Model Insights

This page outlines the analytical logic and evolution behind AI-Bet v3.6 → v5.0, tested across 30 verified races in AU + NZ. Each update refines data-weighting for bias, tempo translation, and endurance mapping.

🧩 1. Core Algorithm Logic

Barrier Position Weighting: Early bias still favours inside draws (1–4) but v5.0 introduces adaptive correction for track shape and rail position. Outside barriers now adjust dynamically by up to ±4.

Weight Delta Adjustment: v5.0 refines weight-based stamina scaling: −1.5 kg = +6 endurance bonus; +2 kg = −4 bias penalty. Applies more strongly on Soft 6–Heavy 8 tracks.

Tempo Translation Ratio: Tracks early-to-late-speed endurance. Horses maintaining ≥ 95 % mid-tempo receive a +7 consistency coefficient.

Track Bias Memory: Bias registry now aggregates by venue × distance. Good → baseline 1.00; Soft + 0.05; Heavy + 0.15 variance.

⚙️ 2. Edge-Calculation System

Pure Database Mode: Odds removed entirely. Model predicts via internal metrics only.

Dynamic Learning Loop: Under-rated winners (> 2 positions above expectation) increase feature weights × 1.5 × per cycle.

Tempo & Bias Correlation: v5.0 adds a cross-weight term between tempo retention and barrier efficiency (r = 0.43 AU sample).
