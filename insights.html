AI-Bet v3.6 Model Insights

This page explains the analytical framework behind AI-Bet v3.6, used in recent trials across Kilmore, Seymour, Geraldton, and Mountaineer Park. It blends data-driven racing metrics with adaptive machine logic that identifies statistical edges in Australian and US racing circuits.

📊 1. Core Algorithm Logic

Barrier Position Weighting: Early bias favouring inside draws (1-4) given +5 stamina and early-speed advantage. Outside barriers adjusted down by –3 based on track geometry and start distance.
Weight Delta Adjustment: Horses carrying > 1.5 kg less than last start are boosted +7 points for improved fitness. Horses +1.5 kg heavier are penalised –4 for slower mid-tempo resilience.
Early Speed Tempo vs Late Finish Ratio: Incorporates a “hold-through-tempo” constant that predicts endurance in mid-tempo shifts. Horses sustaining > 95 % speed ratio receive a +6 bias score.
Track Bias Memory: Each track is tagged as Good, Soft, Heavy, or Fast, weighted by local performance memory. Good tracks apply standard coefficients; heavy tracks add variance factor 0.15.

⚙️ 2. Edge-Calculation System

Pure Database Mode: Removes odds and sentiment data. Only internal metrics (tempo, stamina, barrier, and weight) inform outcome — ensuring bias-free projections.
Dynamic Learning Loop: Each race result feeds back into the dataset. When a horse with low rating outruns model expectation > 2 positions, its feature weights increase by +1.5×.

  • Dominant Edge: Mid-distance (1200–1600 m) Good tracks
